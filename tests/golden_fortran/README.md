# FORTRAN reference output

These OFF files are the authoritative reference for the Python translation.
Generated by compiling and running the original FORTRAN simulator (`13.f90`)
on `examples/seal.txt`.

## Capture command

The exact command — re-run if the parameter file or the FORTRAN source change:

```bash
scripts/build-fortran.sh         # builds .tmp/fortran/toothmaker
scripts/capture-fortran-golden.sh  # runs it, copies output here
```

## Reproducing the run by hand

```bash
gfortran -O2 -ffree-line-length-none -std=legacy \
    -J .tmp/fortran -o .tmp/fortran/toothmaker 13.f90

mkdir -p .tmp/fortran-baseline
.tmp/fortran/toothmaker examples/seal.txt .tmp/fortran-baseline run 100 5
```

(The capture script then renames the trailing-underscore-padded filenames
the FORTRAN produces — e.g. `100_run_______________________.off` —
to clean names like `100_run.off`.)

## Expected shape

- 5 OFF files, one per save block (after 100, 200, 300, 400, 500 iterations).
- Each file: 295 lines total (1 `COFF` header, 1 counts line, 57 vertices,
  236 faces).
- Cell count is stable at 57 from block 1 onwards — cells divide during
  the first 100 iterations from the `Rad=4` initial state of 37 cells, then
  reach steady state for the remainder of the run.
- Z-coordinates grow over time as the epithelium invaginates: ~26 in block 1,
  ~125 in block 5.

## Toolchain

- gfortran (Homebrew GCC 15.2.0_1)
- `-std=legacy` is required because `13.f90` uses fixed-form historical
  conventions (implicit typing, GOTO, single-letter variables).
